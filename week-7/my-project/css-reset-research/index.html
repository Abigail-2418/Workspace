Step 1 — Historical Foundation

Eric Meyer — Reset Reasoning

Eric Meyer created a CSS reset because browsers used to style elements differently.

Things like margins, headings, and spacing could look different depending on the browser.

His reset removed most default styles so developers could start with a cleaner layout.

Problems CSS Resets Solved

  • Different heading sizes
  • Different spacing between browsers
  • Layouts looking inconsistent
  • Default browser styles getting in the way

Step 2 — Modern Evolution

Josh Comeau — Modern CSS Reset

Josh Comeau’s reset feels more modern compared to Eric Meyer’s version.

Instead of deleting basically everything, Josh keeps some useful browser defaults and only fixes the parts that usually cause problems.

His reset also focuses more on accessibility and responsive design.

Differences Between Josh and Meyer

  • Josh uses border-box sizing
  • Josh improves accessibility
  • Josh keeps useful browser defaults
  • Meyer removes almost everything

Critical Questions

Why Reset CSS?

CSS resets help make websites look more consistent across browsers.

Without resets, browsers can apply different default styles that affect layouts and spacing.

Why Use Josh’s Reset Instead of Meyer’s?

Josh’s reset makes more sense for modern web design because browsers are already more consistent today.

His reset improves usability and accessibility without removing everything.

How Do You Use CSS Resets?

CSS resets are usually connected with a link tag inside the head section of HTML.

They can also be imported into another CSS file.

What Is a CSS Boilerplate?

A CSS boilerplate is basically a starter setup for a website project.

It can include resets, helper classes, accessibility fixes, and responsive setup.

Why Use a Boilerplate?

Boilerplates save time because developers do not need to set everything up from scratch.

They can also help keep projects more organized.

Why Is Normalize.css Different?

Normalize.css is different because it does not try to erase every browser style.

Instead, it keeps useful defaults while fixing browser inconsistencies.

My Thoughts

I think CSS resets were probably more important in the past because browsers used to behave a lot more differently.

Josh Comeau’s reset feels more practical because it improves the browser defaults instead of deleting everything.

I also think Normalize.css makes sense because it works with the browser instead of fighting against it.
